# Methodology

## 1. Overview

This document explains the methodology used in the Adaptive AI Tutor Research Lab.

The project follows a full AI/ML pipeline:

data generation → validation → feature engineering → supervised learning → clustering → recommendation → reinforcement learning → evaluation → visualization

The goal is to create a reproducible research prototype for adaptive learning.

## 2. Data Generation

The current version uses synthetic student interaction data.

Each interaction represents one student answering one question.

The generated dataset includes:

- student ID
- question ID
- topic tag
- question difficulty
- elapsed time
- student answer
- correct answer
- correctness label

Synthetic data is used because it allows safe experimentation without exposing real student data.

## 3. Data Validation

Before modeling, the dataset is validated.

The validation step checks:

- required files exist
- required columns exist
- missing values
- answer correctness consistency
- question IDs match between files
- timestamps are ordered correctly

This helps make sure the pipeline does not continue with broken or inconsistent data.

## 4. Feature Engineering

Raw interactions are transformed into machine learning features.

The feature engineering step creates historical learning signals such as:

- student attempt count
- student accuracy so far
- previous answer correctness
- rolling accuracy
- topic-level accuracy
- question-level accuracy
- time since previous attempt
- question difficulty estimate

These features are designed to represent the student's learning history before the current answer.

This is important because using future information would create data leakage.

## 5. Supervised Knowledge Tracing

The supervised learning task predicts whether a student will answer correctly.

The target variable is:

- `is_correct`

The models tested include:

- Dummy Classifier
- Logistic Regression
- Random Forest
- Gradient Boosting

The purpose of this step is to estimate student performance using previous learning behavior and question information.

## 6. Student Clustering

Student clustering groups students based on learning behavior.

The clustering step uses student-level features such as:

- total attempts
- final accuracy
- average elapsed time
- average difficulty score
- topic diversity
- recent accuracy
- accuracy change

The goal is to identify different student learning patterns.

## 7. Recommendation System

The recommendation system creates simple tutor recommendation examples.

Current strategies:

1. Random recommendation  
   Selects a random question.

2. Difficulty-based recommendation  
   Selects question difficulty based on recent student accuracy.

3. Weak-topic recommendation  
   Selects a question from the topic where the student performs weakest.

These are baseline recommendation strategies, not final production-level recommenders.

## 8. Reinforcement Learning

The reinforcement learning component simulates an adaptive tutor.

The tutor observes a student state and selects an action.

State includes:

- recent accuracy
- weak topic
- current step

Actions include:

- easy question
- medium question
- hard question

Reward is based on:

- whether the simulated student answers correctly
- whether the tutor targets a weak topic
- whether the selected question is too easy or too hard

A Q-learning agent is compared with a random policy.

## 9. Evaluation

Different parts of the system are evaluated with different metrics.

Supervised learning metrics:

- Accuracy
- Precision
- Recall
- F1 Score
- ROC-AUC
- Log Loss

Clustering metric:

- Silhouette Score

Reinforcement learning metrics:

- Average reward
- Average correctness
- Policy comparison

These metrics help compare baselines and check whether each component works as expected.

## 10. Visualization

The project creates visual outputs for easier interpretation.

Visualizations include:

- supervised model comparison plots
- student clustering PCA plot
- Q-learning reward curve
- reinforcement learning policy comparison

These plots make the results easier to understand for portfolio readers.

## 11. Demo

The Streamlit demo app presents the main project outputs in an interactive dashboard.

The demo includes:

- dataset overview
- supervised model results
- clustering summary
- recommendation examples
- reinforcement learning comparison

The demo does not train models. It reads the outputs generated by the pipeline.

## 12. Reproducibility

The full pipeline can be run with:

```bash
python run_pipeline.py
```

The smoke test can be run with:

```bash
pytest
```

A successful test confirms that the full pipeline runs and creates the expected outputs.

## 13. Limitations

Current limitations:

- the dataset is synthetic
- student behavior is simulated
- the recommendation logic is simple
- the reinforcement learning environment is simplified
- no real classroom feedback is used
- no long-term learning outcome is measured

These limitations are expected for an early research prototype.

## 14. Summary

The methodology connects multiple AI/ML components into one complete adaptive tutor pipeline.

The project is designed to show not only model training, but also data preparation, evaluation, recommendation logic, reinforcement learning simulation, visualization, testing, and documentation.
